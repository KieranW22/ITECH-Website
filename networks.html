Networks

Network Media

Computer networks can make use of different types of network media. The two types are:

  • Nodes: PC, Server, Switch, Bridge, Router
  • Links: Optic fibers, Coaxial cable, Wireless

Protocols

Protocols are a set of rules that allows devices to communicate over a network. Internet protocols include high level protocols like, TCP, UDP, HTTP and FTP as well as lower-level protocols such as ARP and ICMP.

Analog signals vary continuously and can have many values inside of a range. Digital signals are pulses of voltage (either 1 or 0).

Multiplexing

There are three types of directional transmission:

Simplex
Only one direction
Half-duplex
Both directions but only one direction at a time
Full-duplex
Transmitted in both directions simultaneously

Multiplexing allows multiple signals to be combined into a single channel and transmitted togther. It is mostly used in telecommunications.

Ethernet

Ethernet has been hugely popular due to the its simplicity, ease of integration and because it is inexpensive.

Switching is used in networks to create connections between multiple devices. There are three types of switching:

Circuit
Connection established before transmission. Data follows a dedicated route.
Message
Connection established then broken after transmission. Information is stroed then forwarded to next device.
Packet
Used by Ethernet & Internet. Data broken into smaller packets which can take any route to the destination, where the data is reassembled

Routing

All hosts and interfaces have the same Network ID (prefix) but a unique IP address (suffix). A router is required to connect to other networks as it acts as the defualt gateway for data to pass between networks.

IPv4 uses a 32-bit number represented in dotted decimal to address devices. E.g. 192.168.0.1
IPv6 uses a 128-bit address which is represented in hexadecimal. E.g. 0:0:0:0:0:0:0:1

UDP & TCP

UDP is an unreliable, unordered delivery of packets and provides fragmentation and reassembly, multiplexing and error detection.

TCP is reliable and in-order delivery of packets using all of UDP plus reliable data transfer.

DNS & NAT

DNS allows textual web addresses to be mapped to IP addresses. An example of using DNS would be naviagting to www.google.co.uk in your web browser. You are connecting to an IP server address which is hidden behing Google's URL.

NAT enables private IP addresses to connect to public networks. For example, your home network has one public IP address for all your home devices.
